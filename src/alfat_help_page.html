1. How To Use

  1. Open ALFAT.
  2. Open a file: File > Open File. If no syntax file selected, File > Settings Menu > file settings > Change Syntax and select the syntax file you want from the file loader.
  3. Build flowchart from file: File > Generate Flowchart
  4. Switch between views: File > Flowchart/Splitscreen/Text editor view
  5. Highlight register usage: Registers > (register name)
  6. Highlight broken jump statements: Analysis > Invalid Labels

 

2. Header Buttons

 

File:

Open File: Opens the native file loader will then open allowing you to select the assembly file you want to generate a flowchart for.

Save: Save the current contents of the text editor to the opened file

Save As: Save the current contents of the text editor to a separate file.

Save Flowchart: Save the flowchart displayed in the flowchart window as an image.

Settings Menu: Opens the settings menu.

 

Flowchart:

Generate Flowchart: Generates the initial flowchart from the opened file.

Regenerate Flowchart from Source: Generates the flowchart from the contents of unedited opened file.

Regenerate Flowchart from Editor: Generates the flowchart from the contents of the editor.

 

View:

Text Editor View: Changes the view of the application to the text editor only.

Flowchart View: Changes the view of the application to the flowchart only.

Split Screen View: Changes the view of the application to text editor on the left side and flowchart on the right side.

Reset Zoom: Resets the amount the camera is zoomed to its original view.

 

Analysis:

Partial Tag: Specify where in the opened file you want ALFAT to start the creation of the flowchart.

Registers: Takes a regular expression of registers and highlights the boxes that use them.

Invalid Labels: Highlights all boxes containing invalid labels.

Clear: Clears all highlighting.

 

3. Settings Menu

 

Display Settings:

 

Auto Generate Flowchart:

Yes -> Opens file in text editor and generates the flowchart of opened file upon selection.

No -> Opens file in text editor only (Select Flowchart -> Generate Flowchart to generate)

 

4. Syntax Creation

JSON Format:

{
"keywordPatterns": {
"control": ,
"reserved": ,
"arithmetic": ,
"dataMovement": ,
"register": ,
"commentLine": ,
"constantHex": ,
"constantNumeric": ,
"constantBinary": ,
"constantCharacter":
"doubleQuotedString": ,
"emptySpace": ,
"label": ,
"comment":
}
}

 

Category

Description

Example/Recommended

control

Regular expression for all instructions that change the PC based on a label.

Example: (BR|BRn|BRz|BRp| |JSRR)\\b

reserved

Regular expression for all keywords that specify data types.

Example: (TRAP|GETC|OUT| |.end)\\b

 

arithmetic

Regular expression for all keywords that specify data types.

Example: (NOT|AND| |ADD)\\b

dataMovement

Regular expression for all instructions that load, store or move data.

Example: (LD|LDR| |LEA)\\b

 

registers

Regular expression for all registers that can be referenced by instructions.

Example: "[rR][0-7](,)?"

commentLine

Regular expression for all registers that can be referenced by instructions.

Example: "(;.+?$)|(;)"

constantHex

Regular expression for recognizing hexadecimal values.

Recommended: "[x][A-Fa-f0-9]{1,4}"

constantNumeric

Regular expression for recognizing numeric values.

Recommended: "^[x#]-?[0-9]+|[0-9]+")

constantBinary

Regular expression for recognizing binary values.

Recommended: "\\b[01]+\\b

doubleQuotedString

Regular expression for recognizing strings within quotations.

Recommended: "^(\".*\")_|^([\\\"\\\\].*[\\\"\\\\])"

emptySpace

Regular expression for recognizing empty space.

Recommended: "((?<=\\s)|(?=\\s+))|((?<=,))|(?=\\.)|((?<=\\\")|(?=\\\"))|((?<=\\\\)|(?=\\\\))|((?<=')|(?='))|((?<=;)|(?=;))"

 

label

Regular expression for recognizing labels.

Recommended: "^[a-zA-Z0-9\\-_]+"

comment

the symbol used to denote a single lined comment.

Example: ";
